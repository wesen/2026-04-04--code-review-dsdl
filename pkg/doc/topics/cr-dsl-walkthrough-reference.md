This is the authoritative reference for the walkthrough YAML format. Every walkthrough is a single YAML file. The Go parser (`internal/domain/yaml/parser.go`) reads it and serves it via the REST API. The React frontend renders each step using one of 13 step renderers.

A walkthrough file lives in the walkthroughs directory, which is passed to the server with `-walkthroughs`. The filename (without `.yaml`) is the walkthrough ID used in URLs like `/wt/auth-refactor`.

## Top-level fields

```yaml
# Required fields
id: string
# Unique identifier. Used in URLs: /wt/{id}
# Must be a valid URL-safe string (no spaces, slashes).

title: string
# Human-readable title shown in the page header and in the walkthrough list.

repo: string
# Repository identifier shown in the header metadata.
# Example: "github/acme/backend"

base: string
# Git ref (branch, tag, or commit SHA) for the "before" state.
# Used by source/diff steps to show content from this ref.

head: string
# Git ref for the "after" state. Usually a feature branch.
# Example: "feat/my-feature"

authors: [string]
# List of author names shown in the header metadata.
# Optional. Omit or use empty list if not applicable.

steps: [Step]
# Ordered list of step objects. See Step types below.
# A walkthrough must have at least one step.
```

### Minimal example

```yaml
id: minimal-example
title: "A minimal walkthrough"
repo: github/example/app
base: main
head: feature/my-change
authors: []
steps:
  - type: text
    body: |
      This walkthrough is intentionally minimal.
```

## Step types overview

Each item in the `steps` array is a discriminated union. The `type` field determines which other fields are valid. All steps share one common field:

```yaml
type: string  # Required. One of:
               # text, source, diff, code, compare,
               # link, annotation, checkpoint, reveal,
               # shell, section, branch
```

All other fields are optional unless marked **required**.

---

## text

A prose paragraph. Use for introductions, explanations, and narration between code-heavy steps.

```yaml
- type: text
  id: intro  # Optional. Used by branch steps for goto targets.
  body: |
    This PR replaces the legacy session-based auth with JWT tokens.
    We'll walk through the changes bottom-up, starting at the token
    utility layer.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `body` | string | Markdown-lite prose. Blank lines separate paragraphs. No markdown formatting is rendered — raw text only. |
| `id` | string | Optional identifier. Other steps can link to this via `branch.goto`. |

---

## source

Shows a range of lines from a file at a specific git ref. The backend reads the file from the git repository using the `base` or `head` ref.

```yaml
- type: source
  file: src/utils/token.ts   # Required. Path in the git repo.
  lines: [12, 48]            # Required. [start, end] (1-indexed, inclusive).
  ref: feat/auth-refactor    # Optional. Git ref. Defaults to the walkthrough's `head`.
  highlight: [30, 35]       # Optional. Line range to highlight in the code block.
  note: "New verifyToken helper."  # Optional. Shown below the code block.
```

The file content is fetched from the Go API (`GET /api/files/content`) which reads from the git repository. The ref is required in the URL; if not specified in the YAML, it defaults to `head`.

**Fields:**

| Field | Type | Description |
|---|---|---|
| `file` | string | Path to the file within the git repository. |
| `lines` | [int, int] | `[start, end]` line numbers, 1-indexed and inclusive. |
| `ref` | string | Git ref. Optional. Defaults to walkthrough's `head`. |
| `highlight` | [int, int] | Optional line range to visually highlight. |
| `note` | string | Optional prose note shown below the file badge and code block. |

---

## diff

Shows a unified diff for a specific file between two git refs. The diff is generated by running `git diff A B -- path` in the repository.

```yaml
- type: diff
  file: src/middleware/auth.ts   # Optional. Required if path is not in a diff.
  from: main                     # Required. "Before" git ref.
  to: feat/auth-refactor         # Required. "After" git ref.
  hunks: [2, 3]                  # Optional. Show only these hunk numbers.
  note: "Session lookup replaced with verifyToken call."
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `file` | string | Path to the file in the git repo. |
| `from` | string | "Before" git ref. |
| `to` | string | "After" git ref. |
| `hunks` | [int] | Optional list of hunk numbers to display. If omitted, all hunks are shown. |
| `note` | string | Optional prose note below the diff block. |

---

## code

An inline code snippet. Use for before/after comparisons, short examples, or conceptual illustrations that don't need to be fetched from git.

```yaml
- type: code
  lang: typescript      # Optional. Syntax highlighting hint (typescript, python, sql, etc.).
  body: |
    // Before
    const user = await Session.find(req.cookie.sid);

    // After
    const user = decodeJWT(req.headers.authorization);
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `lang` | string | Optional. Syntax highlighting language hint. |
| `body` | string | Required. The code content. Newlines are preserved. |

---

## compare

Side-by-side view of two file ranges (typically the same path at two different refs). Each side internally uses the `source` renderer.

```yaml
- type: compare
  left:
    file: src/routes/users.ts
    ref: main
    lines: [10, 30]
  right:
    file: src/routes/users.ts
    ref: feat/cursor-pagination
    lines: [10, 38]
  note: "Old offset pagination vs new cursor approach."
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `left` | SourceRef | `{ file, ref, lines }` — the left/before pane |
| `right` | SourceRef | `{ file, ref, lines }` — the right/after pane |
| `note` | string | Optional note above the comparison |

---

## link

An external hyperlink.

```yaml
- type: link
  url: https://jwt.io/introduction   # Required.
  label: JWT primer (external)       # Optional. Shown as link text.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `url` | string | Required. The destination URL. |
| `label` | string | Optional. Defaults to the URL string. |

---

## annotation

An inline comment anchored to a specific file and line. Use for inline review notes: praise, warnings, issues, or informational observations.

```yaml
- type: annotation
  file: src/middleware/auth.ts
  line: 42
  severity: warn   # Required. One of: info, warn, issue, praise
  body: Race condition if token refresh fires mid-request.
```

**Severity meanings:**

- `info` — informational note, no action needed
- `warn` — potential problem the reviewer noticed
- `issue` — bug or incorrect behavior
- `praise` — positive observation

**Fields:**

| Field | Type | Description |
|---|---|---|
| `file` | string | Path to the file in the git repo. |
| `line` | int | Line number (1-indexed). |
| `severity` | string | `info`, `warn`, `issue`, or `praise`. |
| `body` | string | The annotation text. |

---

## checkpoint

An interactive multiple-choice question. The reader selects one option; the renderer shows immediate feedback (correct/incorrect) with an explanation.

```yaml
- type: checkpoint
  prompt: What happens if the token is expired?
  choices:
    - label: Returns 401 Unauthorized
      correct: true
      explanation: The handler checks the expiry before proceeding.
    - label: Falls back to session auth
      correct: false
      explanation: Session auth was removed in this PR.
    - label: Silently ignores the error
      correct: false
      explanation: That would allow unauthenticated requests through.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `prompt` | string | Required. The question text. |
| `choices` | [Choice] | Required. Array of 2–5 Choice objects. |
| `choices[].label` | string | The choice text shown as a button. |
| `choices[].correct` | bool | `true` if this is the correct answer. |
| `choices[].explanation` | string | Shown after the reader selects. Explains why the answer is right or wrong. |

---

## reveal

A collapsible toggle. Use to hide optional deep-dive content — the reader clicks to expand it.

```yaml
- type: reveal
  title: Show edge-case discussion
  body: |
    If the token is expired but the session store is also unavailable,
    the handler falls back to returning 401.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `title` | string | The clickable label. Defaults to "Show more". |
| `body` | string | The content shown when expanded. |

---

## shell

A terminal command block with expected output.

```yaml
- type: shell
  cmd: npm test -- --coverage
  output: |
    PASS  src/auth.test.ts
    coverage: 87.3%
  note: All auth tests pass with good coverage.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `cmd` | string | The command to display. |
| `output` | string | Optional. The expected output. |
| `note` | string | Optional note shown below the block. |

---

## section

A named group of sub-steps. Sections can be nested recursively. The renderer shows a collapsible heading with the sub-steps rendered beneath it.

```yaml
- type: section
  title: Migration notes
  id: migration
  steps:
    - type: text
      body: All clients must update to use `cursor` instead of `page`.
    - type: annotation
      file: src/routes/users.ts
      line: 15
      severity: info
      body: Deprecation warning header added here.
```

**Fields:**

| Field | Type | Description |
|---|---|---|
| `title` | string | Required. The section heading text. |
| `id` | string | Optional. Can be used as a goto target. |
| `steps` | [Step] | Required. Ordered list of child steps. |

---

## branch

A navigation prompt. Presents the reader with multiple labeled options that jump to different sections of the walkthrough.

```yaml
- type: branch
  prompt: What would you like to explore next?
  options:
    - label: Explore the tests
      goto: tests
    - label: Read the architecture doc
      goto: arch
    - label: Jump to the summary
      goto: summary
```

`goto` targets any step with a matching `id`. Steps without explicit IDs get auto-generated IDs based on their index.

**Fields:**

| Field | Type | Description |
|---|---|---|
| `prompt` | string | The prompt shown above the options. |
| `options` | [BranchOption] | Required. Array of 2–5 options. |
| `options[].label` | string | The button text. |
| `options[].goto` | string | The `id` of the target step. |

## TypeScript type (source of truth)

The TypeScript discriminated union is in `frontend/packages/cr-walkthrough/src/types.ts`:

```ts
export type Step =
  | TextStep
  | SourceStep
  | DiffStep
  | CodeStep
  | CompareStep
  | LinkStep
  | AnnotationStep
  | CheckpointStep
  | RevealStep
  | ShellStep
  | SectionStep
  | BranchStep;
```
